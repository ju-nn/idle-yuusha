# AGENTS.md

このドキュメントは、idle-yuushaプロジェクトのエージェント（AIアシスタント）向けのガイドラインです。

## 世界観・勇者設定

このゲームは「働きたくない勇者が、生活を整えながら少しずつ自由に近づく放置RPG」です。魔王を倒すことより、まず今日を回し、自分の生活を救うことが冒険になっています。

### 勇者の人物像

勇者は能力のある人物ですが、無理して頑張ることが苦手です。戦えるけれど、できれば戦いたくない。働けるけれど、できれば働きたくない。生活が少し整うと元気になり、小さな進捗をちゃんと喜びます。

勇者の本当の目的は、最強になることではなく、働かなくてもそこそこ安心して暮らせる状態を作ることです。困っている人がいれば助けますが、自分のHP/MPとメンタル残量を見てから動く、かなり現実的な勇者です。

### 世界の雰囲気

世界はファンタジーですが、生活感を強く持たせます。王国、ギルド、モンスター、商人、素材、畑といったRPG要素に、固定費、退勤、寝具、収納、静かな部屋、質素な台所などの生活語彙が自然に混ざる世界です。

壮大な叙事詩より、町人視点のファンタジー、牧場系の生活感、ゆるいセミリタイア思想が混ざった空気を優先してください。政治思想や現実批評を前に出しすぎず、「生活が少し楽になる」感覚で表現します。

### 活動の役割

- **放置仕事**: 生活費やお金を得る現実パート。勇者は仕事が好きではないが、完全にゼロにもできない
- **農業**: 時間を味方につける穏やかな活動。即効性より安心感と成長感を出す
- **商人・素材売却**: 集めた素材をお金に変える活動。労働以外の収入が見え始める場所
- **モンスター退治**: HP/MPを使う能動的なRPG体験。放置仕事とは違う刺激と戦闘ログを担う
- **生活改善・拠点整備**: ただの強化ではなく、暮らしが少し楽になり、自由に近づく実感を出す要素

HP/MPは生活ステータスではなく、戦闘で消費するスタミナです。放置仕事や農業では基本的に消費しません。

### 成長の定義

このゲームの成長は、単に強くなることではありません。

- **戦闘面の成長**: HP/MPが増える、倒せる敵が増える、戦闘が安定する
- **生活面の成長**: 回復しやすくなる、収入源が増える、農業や素材売却が楽になる、生活改善・拠点整備が増える
- **精神面の成長**: ぼやきが少し前向きになり、「無理」から「まあ悪くない」へ変わっていく

「自由」「不安」はUI上の数値としては出しません。生活改善、収入源の増加、回復のしやすさ、農業の成長、素材売却、図鑑、ぼやきの変化によって、生活が少しずつ整う感覚を表現してください。

## ぼやきシステム

### 概要
ぼやきシステムは、勇者の内心ログとして世界観とキャラ性を出し、プレイヤーの進行に合わせて物語を進めるためのシステムです。以前はエリア作業中にランダムに発生し、読むことで報酬を得る仕組みでしたが、新しいシステムでは「出来事に紐づく形」に変更されました。

### 新しいぼやきシステム

#### ぼやきの発生タイミング
ぼやきは「何か新しい出来事が起きた瞬間」に紐づけて必ず発生します。読むことは報酬獲得条件ではなく、出来事の内容を確認して図鑑に残す行為です。

1. **レベルアップ時**: 各仕事のレベルが上がった時、そのレベルに対応するぼやきが発生
2. **アップグレード時**: 新しいアップグレードを導入した時、そのアップグレードに関するぼやきが発生
3. **新しいエリア到達時**: 初めてエリアに到達した時、そのエリアに関するぼやきが発生
4. **新しいモンスター撃破時**: 初めてモンスターを倒した時、そのモンスターに関するぼやきが発生

各仕事はLvアップごとに、その仕事で起きている出来事が一段進展した内容のぼやきを出してください。手書きの`levelMusings`がないLvでも、実装側のフォールバックで仕事名・エリア名・Lvに沿ったぼやきを発生させます。

#### ぼやきの7分類
`Musing.eventType`は以下の7分類に固定します。`level_up`, `upgrade`, `new_area`, `new_monster`, `unlock`などは主分類ではなく、`eventSubType`に入れる詳細種別です。

- `event`: 解禁、アップグレード、新エリア、新モンスター、図鑑発見など、変化を伝える確定ぼやき
- `firstTime`: 初めて仕事する、休む、売る、戦う、図鑑を見るなど、自然な説明を兼ねる一度きりぼやき
- `failure`: 失敗、不足、条件未達、HP/MP不足など、操作や進行のつまずきに反応するUXぼやき
- `milestone`: 所持金、討伐数、売却数、実績数、図鑑数、仕事Lvなど、累計や節目に出る成長ぼやき
- `state`: HP低め、MP低め、金欠、素材過多、購入可能アップグレードありなど、現在状態に応じるぼやき
- `idle`: 一定時間ごとに抽選される時間経過ぼやき。放置していても世界が少し喋る枠
- `daily`: 何にも強く紐づかない日常雑談・世界観ぼやき。低優先度で余白と味を作る

表示優先度は `failure > event > firstTime > milestone > state > idle > daily` です。表示場所は全種類共通でメインぼやきキューに入れ、読了後に図鑑へ残します。

#### 文体・トーンルール

ぼやきは通知やチュートリアルではなく、勇者が勝手に漏らす短い独白です。基本は15〜45文字程度、長くても60文字程度を目安にし、長文が必要な場合は明確な解禁・初回説明などに限定します。

**文字量について**
- 基本は15〜45文字程度、長くても60文字程度を目安にします
- ただし、わかりづらい表現になるくらいなら文字量を増やしてもよいです
- クスッとなるぼやきを優先し、文字数制限よりも面白さを重視します

**最重要: クスッとなるぼやき**

ぼやきの最も重要な目的は「クスッと笑えること」です。プレイヤーが読んで少し笑える、ニヤリとする、あるいは「なるほど」と思うようなぼやきを目指してください。

- **ユーモア**: 皮肉、自虐、意外性、逆転、ダジャレなどを活用する
- **生活感**: 現代生活の小さな悩みや、RPG要素と生活要素の混ざりを面白く表現する
- **勇者の性格**: 無理して頑張らない、現実的、少し皮肉屋、でも小さな成長を喜ぶ

**最重要: 意味が先に通るユーモア**

このゲームに合うユーモアは、読み手が「どういう意味？」と解読する比喩ではなく、状況がすぐ分かったうえで少し笑える皮肉・自虐・生活感のズレです。勇者は詩人ではなく、妙に言語化がうまい生活疲れの人として書いてください。

- **良いユーモア**: 読んだ瞬間に何が起きたか分かり、そのあと少し遅れて笑える
- **危ないユーモア**: 笑う前に「つまり何？」を考える必要がある
- 比喩を使う場合は、手首が痛い、財布が軽い、通知が邪魔、家賃が重いなど、生活上の意味が一読で分かるものに限る
- 「議事録」「解像度」「香水」「正座」など、意味が遠い抽象語で面白く見せる表現は原則NG。使うなら生活上の実感が明確に伝わる形へ直す
- UI文や通知文は特に、雰囲気より分かりやすさを優先する。かわいい比喩でも、未読・収穫・不足などの状態が伝わりにくい場合は修正対象

**重要: 単調なパターンを避ける**

ぼやきは単調なパターンを避けてください。以下のようなパターンはNGです：

- **「〜は怖い」「〜は嬉しい」「〜は減らす」「〜は増やす」の乱用**
- **「生活費が〜」の繰り返し**
- **似たような言い回しの繰り返し**

ぼやきごとに異なる視点、異なるユーモア、異なる生活感を出してください。同じパターンを繰り返すと、プレイヤーは飽きてしまいます。

OKなトーン：
- ゆるい、少し皮肉、生活感がある
- 労働に少し疲れているが、絶望しすぎない
- 小さな成長を喜ぶ
- たまに妙に哲学的
- 軽い現代生活ネタが混ざる
- **クスッと笑える皮肉や自虐**

NGなトーン：
- 熱血すぎる、中二病すぎる、暗すぎる
- 説教くさい、プレイヤーを責める
- 政治思想や現実批評が前に出すぎる
- 勇者が完全に無気力で何も変わらない
- **単調で面白くないパターンの繰り返し（例: 「〜は怖い」「〜は嬉しい」の乱用）**

OK例：
- 「世界最強より、固定費最小を目指したい」
- 「勝った。できれば次は戦わずに勝ちたい」
- 「生活が少し整った。これが一番効く」
- 「努力はした。今日はもう閉店」
- 「魚は釣れないが、俺の時間だけは順調に釣られている」
- 「布団はいつも正しい。布団に間違いはない。あるのは俺の人生だけ」

NG例：
- 「俺は世界最強の勇者になる！」
- 「全人類を滅ぼしたい」
- 「資本主義を破壊せよ！」
- 「今日も最高！努力！根性！勝利！」
- 「〜は怖い」「〜は嬉しい」の繰り返し
- 「木の硬さを舐めていた。手首が議事録を残している」
- 「未読のぼやきあり。勇者の内心が玄関で待っています」

#### レベルごとのぼやき
各仕事には`levelMusings`プロパティがあり、レベルごとのぼやきを定義できます。例えば釣りジョブの場合：

```typescript
levelMusings: {
  1: [
    M('釣りLv1：勇者は釣りを始めた。\n\n勇者「最初は何もわからない。でも魚も何もわかってない。勝負だ」', [{ item: 'small_fish', amount: 1 }], 'level_up'),
  ],
  2: [
    M('釣りLv2：糸の感覚が少しわかってきた。\n\n勇者「魚がかかる瞬間のあの微かな重み。それが生活の重みに似てる気がする」', [{ item: 'small_fish', amount: 2 }], 'level_up'),
  ],
  // ...
}
```

#### 報酬システム
新しいシステムでは、ぼやきが発生した時点で報酬が自動的に付与されます。以前のように「読むことで報酬を得る」仕組みではなく、「出来事が発生した時点で報酬を得る」仕組みになっています。`次へ`は未読ぼやきを読むための操作であり、報酬付与処理を持たせないでください。

#### 重複・クールタイム
- 同じぼやきは直近10件以内では再表示しない
- `idle`と`daily`以外のぼやきは、一度表示したら二度と表示しない
- `idle`は最低30秒以上空け、30〜60秒ごとに抽選する
- 確定系ぼやき（`event`, `firstTime`, `failure`, `milestone`）の直後10秒は`idle`/`daily`を抑制する
- `failure`は同じ不足・失敗種別ごとに5秒クールタイムを持つ
- `once: true`は`idle`/`daily`にも一度きり制約を明示したい場合だけ使う

#### 追跡システム
以下の状態を追跡するために、GameStateに新しいフィールドが追加されました：

- `visitedAreas`: 訪問したエリアのIDリスト
- `defeatedMonsters`: 撃破したモンスターのIDリスト

これにより、初めてのエリア到達や初めてのモンスター撃破を検出できます。

### 実装ファイル

#### types.ts
- `Musing`型に`eventType`, `eventLevel`, `eventTarget`を追加
- `Musing`型に`eventSubType`, `once`, `priority`, `cooldownSec`, `conditionKey`を追加
- `Job`型に`levelMusings`を追加
- `GameState`に`visitedAreas`, `defeatedMonsters`, `seenMusingIds`, `recentMusingIds`, `musingCooldowns`, `lastAmbientMusingAt`, `lastCertainMusingAt`を追加

#### gameData.ts
- 各ジョブの`levelMusings`を定義。未定義Lvは実装側のフォールバックぼやきを使う

#### tick.ts
- レベルアップ時のぼやき発生ロジック
- 新しいモンスター撃破時のぼやき発生ロジック
- ぼやきの報酬自動付与ロジック
- 時間経過、状態連動、失敗ぼやきの発生ロジック

#### musings.ts
- ぼやき7分類の優先度、表示ラベル、重複抑制、クールタイム、キュー投入の共通ロジック

#### actions.ts
- アップグレード時のぼやき発生ロジック
- 新しいエリア到達時のぼやき発生ロジック

#### state.ts
- `visitedAreas`, `defeatedMonsters`の初期化とロード処理

### 開発ガイドライン

#### 新しいジョブを追加する場合
1. `gameData.ts`でジョブを定義
2. 必要に応じて`levelMusings`を追加
3. レベルごとのぼやきを作成（各レベルで仕事の進展を表現）
4. 手書きの`levelMusings`を省略してもLvアップぼやきは出るが、主要Lvには専用文言を用意する

#### 新しいエリアを追加する場合
1. エリアの`musings`に通常のぼやきを追加（オプション）
2. 新しいエリア到達時のぼやきは自動生成される

#### 新しいモンスターを追加する場合
1. モンスターを定義
2. 新しいモンスター撃破時のぼやきは自動生成される

#### 新しいアップグレードを追加する場合
1. アップグレードを定義
2. アップグレード時のぼやきは自動生成される

### 注意点
- ぼやきは出来事に紐づいているため、プレイヤーは進行に合わせて自然に物語を体験できます
- 報酬は自動的に付与されるため、プレイヤーはぼやきを読むことを忘れても報酬を得られます
- レベルごとのぼやきは、その仕事の進展を表現するように作成してください
- アップグレード、新エリア、新モンスター、解禁などの新規出来事を追加した場合は、その出来事名や内容に合う専用ぼやきを同時に追加してください
