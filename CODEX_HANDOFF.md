# Codex向け引き継ぎメモ

このプロジェクトは「勇者は働きたくない。」の現行プロトタイプです。
仕様書から作り直さず、この実装をベースに改善してください。

## 重要方針

- この時点では、既存の体験を劣化させないことを最優先にする。
- 世界観は「魔王討伐より生活改善」。働きたくない勇者が、生活を整えながら少しずつ自由に近づく放置RPGとして扱う。
- 勇者は熱血主人公ではなく、能力はあるが無理して頑張るのが苦手。世界を救う前に自分の生活を救いたい。
- ぼやきノベルがゲームの核。常に目立つ場所に残す。
- ぼやきは自動で流さない。「次へ」で読む。
- ぼやき報酬は読む操作ではなく、Lvアップ・新エリア・新モンスター・アップグレードなどの出来事発生時に付与する。
- 報酬つきぼやきは、その出来事で何が起きたか分かる内容にする。
- ぼやきは短く、ゆるく、少し皮肉で生活感のある勇者の内心ログ。熱血、暗すぎる表現、説教、政治思想の前面化は避ける。
- ユーモアは「意味が先に通る」ことを最優先する。笑う前に「どういう意味？」と解読が必要な比喩は修正対象。勇者は詩人ではなく、妙に言語化がうまい生活疲れの人として書く。
- Melvor Idle 風に、左メニューから仕事を選ぶ構造を維持する。
- 未解禁の仕事は `？？？` 表示にする。
- HP/MPは生活ステータスではなく、戦闘用スタミナ。
- 「自由」「不安」はUI数値として復活させない。生活が整う感覚として表現する。
- 商人は放置仕事。素材を自動売却してGに変換する。
- モンスター退治は通常放置と違う体験にする。戦闘ログを残す。
- 拠点整備は素材の使い道。HP/MP上限を増やす。
- 発見物・ぼやき・実績は図鑑に統合する。
- 中毒性・気持ちよさはリリース品質の重要要件。詳細は `ADDICTIVE_DESIGN.md` を参照する。
- Cookie Clicker 的な「常に増える」「次目標が見える」「購入が複利になる」「たまに予想外が起きる」構造を、生活が少し楽になる感覚へ翻訳して実装する。
- 焦らせる、取り逃しで罰する、読むことを義務化する方向ではなく、戻ってきた時に成果が積もっていて嬉しい方向で中毒性を作る。

## 開発ツールメモ

- CodeGraph を初期化済み。大きな変更後は `npm run codegraph:sync`、状態確認は `npm run codegraph:status` を使う。
- CodeGraph のDBは `.codegraph/codegraph.db` に生成されるが、ローカル生成物なのでコミットしない。
- AI生成ドット絵を追加する場合は、`PIXEL_ART_GUIDE.md` のルールに従い、生成後に Sprite Fusion Pixel Snapper でピクセル補正してから手修正する。
- Codex の作業手順を増やしたい場合は、公式の [openai/skills](https://github.com/openai/skills) を参照する。必要な skill だけ入れ、プロジェクト本体には汎用 skill 集を丸ごと持ち込まない。

## 改良すると良さそうな箇所

- 実績解除時の小さなトースト表示
- 商人の売却先や相場イベント
- 戦闘で敵HP/勇者攻撃力の概念を追加
- ぼやきの種類と解禁専用ぼやきの追加
- 図鑑のフィルタリング
- セーブデータのバージョン移行
- 「あと少しで買える」拠点整備や解禁先の表示
- 長時間放置後の帰還演出
- レアドロップ、完熟収穫、良い状態イベントなどの小さなサプライズ
