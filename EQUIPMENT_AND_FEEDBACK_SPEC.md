# 装備ドロップ・報酬フィードバック仕様

この仕様は「放置で素材を集め、商人でGに変え、冒険で宝箱を拾い、宝箱開封で装備を得て、生活効率が少し上がる」ループをゲームの中核にするためのものです。

目指す体験は、本格ハクスラではなく「放置して戻ったら宝箱が溜まっていて、開けると変な良装備が出て少し笑える」です。
装備はプレイヤーを厳選労働に沈めるためではなく、暮らしを少し楽にするための複利として扱います。

## 採用する基本ループ

1. 放置仕事で素材とGを少しずつ得る
2. 商人で素材や戦利品をGに変える
3. Gを使って冒険へ出る、または拠点整備を買う
4. 冒険・戦闘・一部放置仕事で宝箱が低確率で落ちる
5. 宝箱を開けると装備が出る
6. 装備で作業速度、素材獲得、売却額、戦闘安定などが少し上がる
7. 次の放置・売却・冒険が少し楽になる

このループの価値は「戦闘のために生活する」ではなく「生活を楽にするために少し冒険する」ことです。

## MVP範囲

最初に入れる装備仕様は、以下に絞ります。

- 装備部位は3つ
  - 武器
  - 防具
  - お守り
- 各部位に1つだけ装備できる
- 装備にはレアリティ、名前、説明、ランダムオプションがある
- ランダムオプションは1から3個
- 不要装備は商人で売却できる
- 装備は直接落ちず、必ず宝箱から出る
- レジェンダリー開封時は専用ぼやきと専用SEを出す
- 装備は図鑑に登録される

この段階では、強化、合成、ソケット、セット効果、耐久度は入れません。

## レアリティ

レアリティは5段階にします。

| レアリティ | 役割 | オプション数 | 入手感 |
| --- | --- | ---: | --- |
| コモン | 序盤の小さな更新 | 1 | まあ拾った |
| アンコモン | 体感できる改善 | 1-2 | 少しうれしい |
| レア | ビルドの方向が見える | 2 | ちゃんと当たり |
| エピック | しばらく使える | 2-3 | かなり当たり |
| レジェンダリー | 名前と効果で記憶に残す | 3 | ぼやき対象 |

ドロップ率の初期案:

| レアリティ | 重み |
| --- | ---: |
| コモン | 70 |
| アンコモン | 22 |
| レア | 6 |
| エピック | 1.7 |
| レジェンダリー | 0.3 |

レジェンダリーは頻繁に出さず、出た時にログ、SE、ぼやき、図鑑でちゃんと祝います。

## 部位ごとの役割

### 武器

戦闘と能動的な冒険に寄せます。

- 攻撃力
- 戦闘速度
- モンスター素材獲得
- 冒険成功率
- HP/MP消費軽減

### 防具

安定、回復、失敗軽減に寄せます。

- 最大HP
- 最大MP
- HP回復速度
- MP回復速度
- 失敗率軽減
- 被ダメージ軽減

### お守り

生活効率、商人、農業、レア運に寄せます。

- 放置仕事速度
- 特定仕事速度
- 素材獲得量
- 売却額
- 農業成長速度
- レアドロップ率
- ぼやき発生率

「全体速度アップ」は強すぎるため、原則レジェンダリー限定にします。

## ランダムオプション

オプションは対象を分け、ひとつの最適解に寄りすぎないようにします。

初期候補:

| オプション | 対象 | 初期値幅 |
| --- | --- | ---: |
| 作業速度 | 特定ジョブまたは全放置仕事 | +3%から+18% |
| 素材獲得 | fish/wood/ore または全素材 | +4%から+20% |
| 売却額 | 商人売却 | +3%から+15% |
| 攻撃力 | 戦闘 | +4%から+22% |
| 防御力 | 戦闘 | +4%から+20% |
| HP回復速度 | 自然回復 | +4%から+18% |
| MP回復速度 | 自然回復 | +4%から+18% |
| 冒険費用軽減 | 冒険出発コスト | -3%から-12% |
| レアドロップ率 | 装備・希少素材 | +2%から+10% |

値幅はレアリティで補正します。
序盤に速度+15%が出ても楽しいですが、全体速度+15%は壊れやすいので避けます。

## 装備名と説明

装備は数値だけでなく、名前と説明でこのゲームらしさを出します。
勇者は強くなりたいのではなく、できれば働かずに生活を安定させたい人です。

名前の方向性:

- RPG道具と生活道具を混ぜる
- 労働、退勤、固定費、休憩、腰、通知、家賃などの生活語彙を使う
- 読んだ瞬間に意味が分かり、少し笑えるものにする

例:

| レアリティ | 名前 | 部位 | 説明 |
| --- | --- | --- | --- |
| コモン | ちょっとマシな木の棒 | 武器 | 棒よりは強い。棒という事実は残る。 |
| コモン | 安売りの作業手袋 | 防具 | 手首への謝罪としては安い。 |
| アンコモン | 商人のそろばん | お守り | 数字が増えると、現実も少し許せる。 |
| レア | 定時退社の短剣 | 武器 | 早く終わらせる意思だけは鋭い。 |
| レア | 腰にやさしいベルト | 防具 | 勇者より先に腰を守る判断。 |
| エピック | 休日を守る盾 | 防具 | 予定を入れようとする世界を少し弾く。 |
| レジェンダリー | 有給の聖剣 | 武器 | 抜いた瞬間、今日は休んでもいい気がする。 |
| レジェンダリー | 不労のマント | 防具 | 羽織ると労働意欲が静かに消える。 |
| レジェンダリー | もう働きたくない指輪 | お守り | 願いが強すぎて、効果欄が少し震えている。 |

## 入手経路

装備は主に宝箱開封で手に入ります。
宝箱は主に冒険で手に入り、生活系の宝箱は低確率で放置仕事からも出ます。

| 入手元 | ドロップ方針 |
| --- | --- |
| 冒険 | 宝箱の主な入手元。武器、防具、お守りすべて出る |
| 戦闘 | 宝箱が出やすい。武器、防具寄り |
| 商人 | 生活の宝箱がまれに出る。売却系装備寄り |
| 木こり/採掘/釣り | 古びた宝箱がまれに出る。仕事専用の生活装備寄り |
| 農業 | 生活の宝箱がまれに出る。お守り、回復系装備寄り |

放置仕事の宝箱ドロップは低確率にし、冒険の価値を残します。

## 宝箱と開封

宝箱は帰還導線の中心です。
プレイヤーは放置後に「素材が増えた」だけでなく「宝箱が溜まっている」ことを楽しみに戻ってきます。

初期宝箱:

| 宝箱 | 主な入手元 | 中身の方向性 |
| --- | --- | --- |
| 古びた宝箱 | 低確率の通常仕事 | コモン中心 |
| 生活の宝箱 | 商人、農業、生活寄り行動 | お守り、生活効率寄り |
| 冒険者の宝箱 | 冒険、戦闘 | 戦闘・冒険向け |
| 城跡の宝箱 | 終盤冒険 | レア以上が少し出やすい |
| まばゆい宝箱 | 超低確率 | 高レア寄り |

開封時の方針:

- 開封はプレイヤー操作で行う
- 1個ずつ開封できる
- 溜まりすぎた時のために、まとめて開封もできる
- 開封結果は短い演出トーストで表示する
- レア以上、特にレジェンダリーはSEとぼやきを強める
- 開封ログは図鑑に残す

開封は課金ガチャではなく、放置成果の受け取り演出です。
射幸心を使いすぎず、「戻ってきた自分への小さなご褒美」として扱います。

## 売却と所持上限

装備管理は重くしすぎません。

- 初期所持上限は30個
- 拠点整備で上限を増やせる
- 装備は1クリックで売却できる
- 売却額はレアリティとオプション数で決まる
- 現在装備中のものは売却前に確認する

自動売却はMVPでは入れません。
ただし、将来的に「コモン自動売却」は追加候補にします。

## 比較UIの方針

装備比較は「強いかどうか」を完全自動判定しすぎず、でも迷わせすぎない形にします。

- 現在装備との差分を表示する
- 速度、獲得量、売却額などの主要効果は色で増減を見せる
- 同じ部位だけ比較する
- 装備ボタン、売却ボタン、ロックボタンを用意する

戦闘特化装備と生活特化装備は単純比較できないため、「総合力」だけで判断させません。

## 装備とぼやき

装備はぼやきシステムと接続します。

発生させるぼやき:

- 初めて装備を拾った
- 初めて装備した
- 初めてレア以上を拾った
- 初めてレジェンダリーを拾った
- 装備売却を初めて行った
- 装備で作業速度が目に見えて上がった

`Musing.eventType` は主に `event` または `firstTime` を使い、詳細は `eventSubType` に入れます。

例:

- `eventType: 'event'`
- `eventSubType: 'equipment_drop'`
- `eventTarget: 'unpaid_overtime_ring'`

レジェンダリー用ぼやき例:

> 勇者「不労のマントを拾った。名前だけでMPが少し回復する。効果より先に思想が強い」

## SEと報酬演出

SEはこのゲームの触り心地を支える重要要素です。
BGMより先に、短く気持ちいいSEを整えます。

初期SE:

| 場面 | 音の方向性 | 備考 |
| --- | --- | --- |
| ゲージ完了 | チャリン | 最も頻繁。短く軽く |
| 複数報酬 | チャチャリン | 通常より少し嬉しい |
| 売却成功 | チャリリリン | Gが増えた感 |
| レア素材 | キラリン | 控えめな発見音 |
| 装備ドロップ | キラーン | 通常レアより少し強い |
| レジェンダリー | 専用ジングル | 1秒以内 |
| 冒険出発 | ポン/シュッ | 軽い出発感 |
| 冒険成功 | 小ファンファーレ | 派手すぎない |
| 失敗 | コツン/しょぼん | 責めない音 |

SEは「鳴ったら得をしたと分かる」ことを優先し、長すぎる音は避けます。

## ゲージ完了テンポ

序盤の1ゲージは短めにします。

- 序盤: 3から8秒
- 中盤: 5から12秒
- 終盤: 8から20秒

ゲージが長くなる場合は、途中で別の報酬やログが動くようにします。
プレイヤーが何も起きていないと感じる時間を減らします。

## 連続報酬

同じ作業でも、たまに報酬が跳ねる瞬間を作ります。

例:

- 通常: 木材 +1、チャリン
- 小当たり: 木材 +2、チャチャリン
- 中当たり: 木材 +3、きれいな枝 +1、チャチャチャリン
- レア: 装備または希少素材、キラリン

確率の初期案:

| 結果 | 確率 |
| --- | ---: |
| 通常 | 82% |
| 小当たり | 13% |
| 中当たり | 4% |
| レア | 1% |

レア枠の中で、素材、ぼやき、装備などに分岐します。

## 音量と設定

音は必ず設定で制御できるようにします。

- SEオン/オフ
- SE音量
- 連続SEの簡略化
- 初期音量は控えめ
- ブラウザの自動再生制限に合わせ、初回操作後に音を有効化する

放置ゲームは長時間開かれるため、音疲れを避けます。

## 実装データ案

型の方向性:

```typescript
export type EquipmentSlot = 'weapon' | 'armor' | 'charm';
export type EquipmentRarity = 'common' | 'uncommon' | 'rare' | 'epic' | 'legendary';

export type EquipmentOption = {
  id: string;
  target:
    | 'jobSpeed'
    | 'resourceGain'
    | 'sellPrice'
    | 'attack'
    | 'defense'
    | 'hpRegen'
    | 'mpRegen'
    | 'adventureCost'
    | 'dropRate';
  scope?: string;
  value: number;
};

export type EquipmentItem = {
  uid: string;
  baseId: string;
  name: string;
  description: string;
  slot: EquipmentSlot;
  rarity: EquipmentRarity;
  options: EquipmentOption[];
  obtainedAt: number;
  locked?: boolean;
};
```

`GameState` 追加候補:

```typescript
equipmentInventory: EquipmentItem[];
equipped: Partial<Record<EquipmentSlot, string>>;
seenEquipmentBaseIds: string[];
equipmentDropLog: string[];
settings: {
  seEnabled: boolean;
  seVolume: number;
  simplifiedComboSe: boolean;
};
```

既存の `items` は素材・発見物用として維持し、装備は個体値があるため別配列にします。

## 実装順

1. SE設定と報酬イベントの発火口を作る
2. ゲージ完了、売却、レア入手にSEを割り当てる
3. 装備型と所持欄を追加する
4. 冒険・戦闘で装備ドロップを追加する
5. 装備画面で装備、比較、売却をできるようにする
6. 装備効果を作業速度、売却額、戦闘へ反映する
7. レジェンダリーぼやき、図鑑、実績を追加する

SEを先に入れるのは、装備が入る前でも報酬感が上がるためです。
装備はその後、ゲームループの中核として広げます。

## 後回しにするもの

- 装備強化
- 装備合成
- セット効果
- ソケット
- 耐久度
- 装備クラフト
- レイド的な高難度コンテンツ
- 厳密なDPSシミュレーション
- 大量の部位

これらは便利ですが、最初に入れるとゲームの焦点が「生活改善」から「装備管理」へ寄りすぎます。

## 成功条件

この仕様が成功している状態:

- 放置後に戻る理由が増えている
- 素材売却と冒険が自然につながっている
- 装備を拾った時、数値だけでなく名前でも少し楽しい
- レア装備が出た時に音、ログ、ぼやきでちゃんと嬉しい
- 装備管理が面倒な作業になっていない
- 生活効率が上がり、「少し楽になった」と感じられる

逆に、装備確認が苦痛になったり、速度装備だけが正解になったり、倉庫整理が労働に見え始めたら調整対象です。
