# ドット絵アートルール

このプロジェクトの最終ビジュアルは、生活感のあるファンタジー放置RPGのドット絵で統一します。
見た目のかわいさより先に、ゲーム中でブレずに読めること、アニメーション時に軸がずれないことを優先します。

## 基本サイズ

ゲーム全体の基準は **32pxタイル** です。

| 用途 | 基準キャンバス | 備考 |
| --- | ---: | --- |
| 小アイコン | 32x32 | 素材、通貨、小さな状態表示 |
| 装備アイコン | 32x32 または 48x48 | UI内では32px表示を基本にする |
| キャラクター | 32x48 | 足元を32pxタイルに合わせる |
| 小型モンスター | 32x32 または 48x48 | 地面接地がある場合はbottom-center |
| UI報酬プロップ | 32x32 | 宝箱、袋、通知用。まず小さく読む |
| 開封/獲得演出 | 32x32 | 宝箱本体は32px以下。光や結果カードで演出する |
| 背景/エリア | 32pxタイル単位 | 32, 64, 96, 128px幅で組む |

## 表示倍率

ドット絵は必ず整数倍で表示します。

- 32px素材: 32 / 64 / 96 / 128px
- 64px素材: 64 / 128px

宝箱、素材、装備などのUI内アイコンは原則32px以下で表示します。
大きく見せたい場合も、まずは周囲の余白、SE、短い光エフェクト、結果テキストで気持ちよさを作ります。

避ける表示例:

- 74px
- 210px
- 1.5倍や1.64倍などの半端な拡大

半端なサイズはピクセルのにじみ、揺れ、線の太さの不安定さにつながります。

CSSでは必ず以下を指定します。

```css
image-rendering: pixelated;
image-rendering: crisp-edges;
```

## アンカー

アニメーション素材は、全フレームで同じアンカーを守ります。

| 種類 | アンカー |
| --- | --- |
| キャラクター | feet / bottom-center |
| モンスター | bottom-center |
| 宝箱などの接地プロップ | bottom-center |
| 浮遊物 | center |
| UIアイコン | center |
| 魔法/光エフェクト | center |

宝箱、キャラクター、モンスターは **bottom-center固定** です。
フタが開いて上に伸びても、箱本体の底辺と中央位置は動かしません。

## アニメーションのズレ防止

アニメーション素材は以下を必須にします。

1. 全フレーム同一キャンバスサイズ
2. 全フレーム同一スケール
3. bottom-center または center のアンカー固定
4. 本体がセル端に触れない
5. 光、星、煙、衝撃波などの大きいエフェクトはできるだけ別レイヤー
6. GIFやスプライトシートをCSSで半端なサイズに拡大しない

悪い例:

- フレームごとに本体の中心が左右へ動く
- 開いたフタや光を含めてトリミングした結果、箱本体が下へずれる
- 光エフェクトがセル端に触れて、全体のクロップ基準が変わる
- 32px素材を半端なサイズに拡大してピクセルがにじむ

## 宝箱ルール

宝箱はこのゲームの帰還導線なので、最初に統一します。

- 開封演出用キャンバス: 32x32
- 一覧表示: 32x32
- 開封演出表示: 32x32
- アンカー: bottom-center
- 箱本体の底辺位置を固定
- 光はCSSまたは別FXで小さく足す
- 宝箱本体アニメは「閉じる、揺れる、開く、開いたまま」を基本にする

レアリティ差は箱本体を増やすより、まず光色で表現します。

| レアリティ | 光色 |
| --- | --- |
| コモン | 白/薄い黄 |
| アンコモン | 緑 |
| レア | 青 |
| エピック | 紫 |
| レジェンダリー | 金 |

## 生成時プロンプトルール

生成する場合は必ず以下を含めます。

- pixel art
- fixed canvas
- same scale across all frames
- stable bottom-center anchor
- body/object remains in same position
- no frame crosses cell edges
- solid #FF00FF background
- no text, no labels, no borders

宝箱のように光が出る素材でも、本体は32x32のまま固定します。
本体アニメと光エフェクトは別にした方が、ズレが少なく、レアリティ差も作りやすくなります。

## 生成後のピクセル補正

AI生成ドット絵は、見た目がドット絵でもピクセルグリッドが微妙に崩れることがあります。
生成直後の素材は [Sprite Fusion Pixel Snapper](https://www.spritefusion.com/pixel-snapper) に通し、mixel、ぼやけ、半端なピクセルを補正してから手修正します。

Pixel Snapper は仕上げツールではなく、下処理として扱います。補正後も以下は必ず目視で確認してください。

- 32px基準のキャンバスに収まっているか
- アンカーが全フレームで揃っているか
- 不要な色数増加や輪郭の欠けがないか
- Asepriteなどで最後に手で直せる状態になっているか

## 実装チェックリスト

ドット絵を追加・更新する時は確認します。

- キャンバスサイズはルール内か
- 表示サイズは整数倍か
- `image-rendering` が指定されているか
- フレーム間でアンカーがズレていないか
- 本体が端に触れていないか
- エフェクトが本体の位置決めを壊していないか
- UI上で文字やボタンを押し出していないか
